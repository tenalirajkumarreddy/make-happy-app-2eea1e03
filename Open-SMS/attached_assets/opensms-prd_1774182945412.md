## 1. What is OpenSMS

OpenSMS is a **zero-infrastructure SMS gateway** that runs entirely on an Android phone.

Install the APK → get a URL → call it from any backend → SMS sent.

No VPS. No Redis. No database. No relay. No Docker. No cloud bill.

The phone **is** the server. The phone **is** the queue. The phone **is** the gateway.

### Who is this for

Developers who want OTP / transactional SMS without paying per message, who already have a spare Android phone or dedicated SIM, and want a single `npm install opensms` or HTTP call integration.

### Primary Use Cases (Priority Order)

| Priority | Use Case |
|---|---|
| 1 | OTP / Phone Auth — replace Firebase Phone Auth |
| 2 | Transactional alerts — orders, payments, shipments |
| 3 | Internal tool notifications — server alerts, ops |
| 4 | Marketing / bulk SMS (carrier fair-use applies) |

## 2. Architecture Decision

### Why NOT a relay server

| Relay Server Model | APK-Direct Model |
|---|---|
| Extra hop: client → relay → APK | Direct: client → APK |
| Needs VPS (~₹500/mo) | Zero infra cost |
| Redis + DB = extra failure points | In-memory = zero failure points |
| 50–300ms overhead | ~5ms overhead (LAN) |
| Complex deploy | Just install APK |

### Chosen Architecture: Embedded HTTP + Coroutine Channel

```
┌─────────────────────────────────────────────────────────┐
│                    Android Phone                         │
│                                                         │
│   ┌──────────────────────────────────────────────────┐  │
│   │              ForegroundService                    │  │
│   │                                                  │  │
│   │   NanoHTTPD           Kotlin Channel             │  │
│   │   :8080          ──►  (in-memory queue)    ──►   │  │
│   │   HTTP Server         capacity: 1000             │  │
│   │                                                  │  │
│   │                       SmsManager                 │  │
│   │                       (native SMS send)          │  │
│   └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
         ▲
         │  HTTP POST /send
         │  Authorization: Bearer <api_key>
         │
┌────────┴──────────┐
│  Client Backend   │
│  (any language)   │
└───────────────────┘
```

### Why NanoHTTPD

- Pure Java library — ~50KB, zero dependencies
- Runs inside any Android app, no special permissions
- Handles concurrent requests with thread pooling
- Starts in milliseconds
- Battle-tested in thousands of Android apps

### Why Kotlin Channel (not a DB queue)

- Pure RAM — microsecond enqueue/dequeue
- Kotlin-native — no serialization, no disk I/O
- Built-in backpressure — `SUSPEND` mode blocks sender if phone is overwhelmed
- **Accepted tradeoff:** queue lost on force-kill — client simply retries on 503

## 3. Communication Flow

### LAN (Same Network) — Fastest Path

```
t=0ms    Client sends  POST http://192.168.1.42:8080/send
t=2ms    NanoHTTPD receives request on phone
t=2ms    API key validated (in-memory string compare)
t=3ms    Template rendered (string replace, no I/O)
t=3ms    SmsJob pushed to Kotlin Channel
t=3ms    HTTP 202 returned to client   ◄── CLIENT DONE
t=4ms    Channel consumer coroutine picks up job
t=4ms    SmsManager.sendTextMessage() called
t=~2s    SMS arrives at target phone
t=~2s    Delivery PendingIntent fires
t=~2s    Webhook POST fired to client (if configured)
```

The HTTP response comes back at **~3ms**. Delivery confirmation comes via webhook.

### Over the Internet (Tunnel)

Same flow, but `t=0` to `t=3ms` becomes `t=0` to `~30ms` due to tunnel RTT. Still sub-50ms — faster than any cloud SMS provider.

The phone has no public IP (carrier NAT). Use one of these to expose port 8080:

```bash

```

### Tailscale (Best for Teams — Private Mesh VPN)

```bash
# Install Tailscale app on phone → sign in
# Phone gets stable IP: 100.x.x.x
# Client connects to: http://100.x.x.x:8080
```

### Ngrok

```bash
pkg install ngrok  # via Termux
ngrok http 8080
# Outputs: https://xxxx.ngrok.io
```

### Static IP SIM

Some carriers offer static IP SIMs (~₹500/mo). Zero software needed.

### 5.6 ForegroundService

```kotlin
class SmsGatewayService : Service() {
    private lateinit var httpServer: OpenSMSHttpServer  // NanoHTTPD subclass
    private val messageChannel = Channel<SmsJob>(
        capacity = 1000,
        onBufferOverflow = BufferOverflow.SUSPEND
    )
    private val messageLog = CircularBuffer<MessageRecord>(maxSize = 500)
    private val stats = StatsCounter()   // atomic, resets at midnight
    private val rateLimiter = TokenBucketRateLimiter(prefs.smsPerMinute)

    override fun onCreate() {
        startForeground(NOTIF_ID, buildNotification())
        httpServer = OpenSMSHttpServer(prefs.port, messageChannel, prefs.apiKey)
        httpServer.start()
        launchQueueConsumer()
    }

    private fun launchQueueConsumer() = scope.launch(Dispatchers.IO) {
        for (job in messageChannel) {
            rateLimiter.acquire()
            SmsSender.send(
                context = this@SmsGatewayService,
                job = job,
                onSent = { messageLog.update(job.messageId, "sent"); stats.incrementSent() },
                onDelivered = { messageLog.update(job.messageId, "delivered"); fireWebhook(job, "delivered") },
                onFailed = { reason -> messageLog.update(job.messageId, "failed", reason); stats.incrementFailed(); fireWebhook(job, "failed") }
            )
        }
    }

    override fun onDestroy() { httpServer.stop(); messageChannel.close() }
}
```

### 5.8 SMS Sender

```kotlin
object SmsSender {
    fun send(context: Context, job: SmsJob, onSent: () -> Unit, onDelivered: () -> Unit, onFailed: (String) -> Unit) {
        val smsManager = context.getSystemService(SmsManager::class.java)

        val sentPI = buildPendingIntent(context, SMS_SENT_ACTION, job.messageId)
        val deliveredPI = buildPendingIntent(context, SMS_DELIVERED_ACTION, job.messageId)

        if (job.body.length <= 160) {
            smsManager.sendTextMessage(job.to, null, job.body, sentPI, deliveredPI)
        } else {
            val parts = smsManager.divideMessage(job.body)
            smsManager.sendMultipartTextMessage(job.to, null, parts,
                ArrayList(parts.map { sentPI }), ArrayList(parts.map { deliveredPI }))
        }
    }
}

// BroadcastReceiver registers for SMS_SENT_ACTION and SMS_DELIVERED_ACTION
// Maps resultCode to: RESULT_OK → callback, else → onFailed(errorReason)
```

**Error mapping:**

| Android Result Code | OpenSMS Error Key |
|---|---|
| `RESULT_ERROR_NO_SERVICE` | `no_service` |
| `RESULT_ERROR_RADIO_OFF` | `airplane_mode` |
| `RESULT_ERROR_NULL_PDU` | `invalid_number` |
| `RESULT_ERROR_GENERIC_FAILURE` | `carrier_error` |

## 7. In-Memory Queue Design

### Kotlin Channel

```kotlin
val messageChannel = Channel<SmsJob>(
    capacity = 1000,
    onBufferOverflow = BufferOverflow.SUSPEND  // backpressure: block HTTP response until space
)
```

### SmsJob

```kotlin
data class SmsJob(
    val messageId: String,      // random 8-char hex ID
    val to: String,             // E.164 phone number
    val body: String,           // Fully rendered body (template already applied)
    val templateName: String?,  // For logs
    val webhookUrl: String?,    // Optional delivery receipt callback
    val enqueuedAt: Long = System.currentTimeMillis()
)
```

### Consumer

```kotlin
scope.launch(Dispatchers.IO) {
    for (job in messageChannel) {
        rateLimiter.acquire()   // token bucket, respects SMS/min config
        smsSender.send(job)     // fires SMS, waits for sent intent
    }
}
```

Single consumer = natural rate limiting + no concurrency issues with SmsManager.

### Overflow Behavior

| Queue Depth | HTTP Response |
|---|---|
| 0–999 | 202 Accepted immediately |
| 1000 (full) | Suspends HTTP request, waits up to 30s for space |
| Still full after 30s | 503 Service Unavailable — client must retry |

### Rate Limiter (Token Bucket)

```kotlin
class TokenBucketRateLimiter(private val tokensPerMinute: Int) {
    private var tokens = tokensPerMinute.toDouble()
    private var lastRefill = System.currentTimeMillis()

    @Synchronized
    suspend fun acquire() {
        refill()
        while (tokens < 1.0) { delay(100); refill() }
        tokens -= 1.0
    }

    private fun refill() {
        val now = System.currentTimeMillis()
        tokens = minOf(tokensPerMinute.toDouble(), tokens + (now - lastRefill) / 60000.0 * tokensPerMinute)
        lastRefill = now
    }
}
```

## 8. Template Engine

### Syntax
Variables: `{{variable_name}}` — Mustache-style, double curly braces.

### Renderer

```kotlin
object TemplateEngine {
    private val VAR_REGEX = Regex("""\{\{(\w+)\}\}""")

    fun render(body: String, vars: Map<String, String>): String {
        val missing = VAR_REGEX.findAll(body).map { it.groupValues[1] }.filter { it !in vars }.toList()
        if (missing.isNotEmpty()) throw MissingVariableException(missing)
        return vars.entries.fold(body) { acc, (k, v) -> acc.replace("{{$k}}", v) }
    }

    fun extractVars(body: String): List<String> =
        VAR_REGEX.findAll(body).map { it.groupValues[1] }.distinct().toList()
}
```

### Validation

- Variable names: `\w+` only (alphanumeric + underscore)
- Max 10 variables per template
- Rendered body ≤ 640 characters
- All `{{vars}}` must be supplied at call time → 400 error if missing

## 9. Security Model

### API Key
- Generated: `UUID.randomUUID().toString().replace("-","")` → 32 hex chars
- Stored: `EncryptedSharedPreferences` (Android Keystore backed)
- Validated: O(1) in-memory string comparison on every request
- Regeneratable from Settings (shown in UI with confirmation dialog)

### Transport
- LAN: HTTP acceptable — traffic never leaves local network
- Internet: Use Cloudflare Tunnel / Tailscale / Ngrok for automatic HTTPS
- Never expose port 8080 directly to public internet without TLS

### Rate Limiting (two layers)
1. HTTP layer (NanoHTTPD): max 100 concurrent connections
2. SMS layer (token bucket): configurable per-minute limit (default: 10/min)

### IP Allowlist (Settings toggle, optional)
- If enabled, requests from non-listed IPs return 403
- Useful for locking gateway to a specific server IP

## 14. Build Order

### Step 1 — Theme
`Color.kt`, `Type.kt`, `Theme.kt` — implement §11 design tokens. Match mockup.

### Step 2 — Data Structures
`SmsJob`, `MessageRecord`, `Template` data classes. `AppPreferences` wrapper.

### Step 3 — Template Engine
`TemplateEngine.kt` + `TemplateRepository.kt`. Unit test all 5 default templates.

### Step 4 — HTTP Server
`OpenSMSHttpServer.kt` — all 4 endpoints + auth middleware + JSON handling.

### Step 5 — SMS Layer
`SmsSender.kt` + `DeliveryReceiver.kt`. **Test on physical device — emulators can't send SMS.**

### Step 6 — Queue
`MessageChannel.kt` + `RateLimiter.kt` + `QueueConsumer.kt`.

### Step 7 — ForegroundService
`SmsGatewayService.kt` — wire all components. Persistent notification. Lifecycle.

### Step 8 — Boot Receiver
`BootReceiver.kt` + AndroidManifest registration.

### Step 9 — UI Screens
In order: Setup → Settings → Dashboard → Templates → Logs.

## 15. Non-Functional Requirements

| Requirement | Target | How |
|---|---|---|
| HTTP → queue latency | < 5ms | In-memory Channel, no I/O |
| Queue → SMS dispatch | < 1ms | Direct coroutine handoff |
| End-to-end (API call → SMS sent) | < 3s | No relay hops |
| APK RAM | < 80 MB | No DB, all in-memory |
| Boot-to-ready | < 5s | ForegroundService starts immediately |
| Queue capacity | 1,000 jobs | Channel capacity |
| Android support | API 26+ | Min SDK 26 |
| APK size | < 10 MB | Minimal deps |
| Concurrent HTTP | 100+ | NanoHTTPD thread pool |
| API key validation | O(1) | In-memory string compare |
| npm bundle | < 50 KB | axios only |
