⚠️ Critical: When Can Order Be Reversed?
  • No Payment: Can reverse completely (stock + balance restored)
  • Partial Payment: Create Sale Return + Refund difference
  • Fully Paid: Full Sale Return + Full Refund OR Credit Note
  • Invoice Issued: Cancel invoice → Reverse Sale → Cancel Order
  • Next Day: Can still reverse with proper audit trail
  • Month-end Closed: Requires admin approval + Credit Note
Reversal Workflow (Step-by-Step):
  1. Detect Error: User realizes delivery was wrong
  2. Check Payment Status: Has payment been recorded?
  3. If No Payment:
    • Create Sale Return (adds stock back, clears outstanding)
    • Change Order status to CANCELLED (not deleted)
    • Cancel/Delete any invoices
  4. If Payment Recorded:
    • Create Sale Return with refund amount
    • Issue Credit Note or process refund
    • Order stays DELIVERED but with return note
  5. Audit Trail: All actions logged with reason and timestamp

✅ Best Practice: NEVER Delete Records
  • Orders: Change status to CANCELLED, don't delete
  • Sales: Create Sale_Return record, don't delete
  • Invoices: Issue Credit Note, don't delete
  • Stock: Reverse via Sale_Return (adds back), don't manually adjust
  • Balance: Reverse via Credit Note, don't manually edit
